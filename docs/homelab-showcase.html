Infrastructure portfolio

Operate the platform, not just tools.

Platform Lab is a hands-on infrastructure portfolio built around Proxmox, Terraform, Ansible, backup and recovery, observability, and a measured path into AWS. It documents the decisions, workflows, and tradeoffs behind a lab environment designed to operate like a real small-team platform.

Storage lane
Warm tier, cold tier, and restore posture
3-2-1 bias

nas01 · TrueNAS Scale

RAIDZ1 storage platform for the PBS warm tier, shared NFS for the monitoring VM, and persistent app data. A storage service, not a general-purpose compute box.

pbs-tank · warm restore tier

The PBS datastore writes directly to a NAS-backed NFS share. This is the local recovery path designed for routine restores and drill evidence capture.

pbs-offsite · cold retention tier

Backblaze B2 extends the chain with lower-cost offsite retention. Weekly sync closes the gap between convenience and disaster resilience.

High availability lane

mon01 is the intentional live-migration workload on shared NFS storage — monitoring survives a node failure without forcing the same dependency onto every VM.

Planned · M7

Monitoring + SLO stack

Prometheus, Grafana, Alertmanager, Loki, and Uptime Kuma arrive as a coherent observability milestone. Four SLOs defined with availability SLIs, dual-window burn rate alerting, and Grafana error budget panels. Logs use a two-layer strategy — local logrotate in the baseline role prevents disk-fill; Promtail ships to Loki for search. Tempo + OpenTelemetry complete the third observability pillar, with trace-to-log correlation in Grafana once cross-system spans exist.

Planned · M7

DNS failover lane

Pi-hole runs across both a VM and bare metal. The value is role portability and failure-domain thinking, not just filtering.

Planned · M6–M8

AWS parity track

A cloud mirror extends the story beyond the local lab and proves the same operational thinking can cross into AWS. Cost is treated as a first-class operational signal — static budget gates pre-spend, Cost Anomaly Detection routed through the same Alertmanager that handles service-down alerts.

Planned · M10

LLM inference capstone

Local inference added as an AIOps-adjacent capstone, giving the lab an alert-triage service that extends the platform beyond core infrastructure and into operator-assistance workflows.

Stack map

Technology lanes

Provision, configure, monitor, protect, and publish.

Provisioning

Terraform across Proxmox, AWS, and NAS definitions, with explicit bootstrap exceptions for the control-plane story.

Configuration

Ansible, Vault, and Bash for reusable role design across VMs, bare metal, and cloud targets.

Runtime

Docker Compose by design, not omission. The project stays close to what a small ops team can realistically run well.

Observability

Prometheus, Grafana, Loki, Alertmanager, Uptime Kuma, Tempo, and OpenTelemetry — metrics, logs, and traces with SLO thinking as the reliability spine.

Recovery

PBS, TrueNAS, NFS-backed warm storage, B2 offsite retention, and restore-drill evidence.

Delivery

GitHub Actions now for public sync, later for the infrastructure delivery pipeline and release discipline.

Reference

Key specs and talking points

Architecture review, interviews, and implementation discussion.

Cluster

2-node Proxmox VE with Corosync QDevice and explicit node-role separation.

Warm backup tier

pbs-tank on a NAS-backed NFS share for fast operational restores.

Cold backup tier

pbs-offsite on Backblaze B2 with 90-day retention behavior.

RTO target

Sub-30-minute full VM restore from PBS warm tier, with drills tracked as reliability evidence.

RPO target

24-hour recovery point — daily VM snapshots to pbs-tank on NAS NFS share.

SLO approach

Four availability SLOs (Pi-hole 99.5%, nginx 99.0%, Prometheus 99.5%, PBS 99.0%) with 30-day rolling windows and dual-window burn rate alerting.

Control host

auto01 becomes the long-lived Ansible and Terraform execution surface.

High availability

mon01 on shared NFS storage — live migration without making every VM depend on that runtime path.

Cloud mirror

AWS us-east-1 with Terraform-managed EC2, VPC, IAM, and CloudWatch — same operational discipline in the cloud.
